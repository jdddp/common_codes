### `python/`

Python 相关工具与主应用代码，主要分为四部分：

- `python/hdy/parse_file/`
  声呐文件查看器与数据解析主工程。支持 `HDY` / `Fishing` 两种协议，自带 GUI、算法管线、伪彩显示、CSV 导出与扇形图映射。
- `python/hdy/parse_HDY/`
  更偏底层或批处理的 HDY / Fishing 解析脚本。
- `python/yolo/`
  与 YOLO 相关的推理封装与数据准备脚本，包括标注转换、推理类以及模型转换等。
- `python/utils/`
  工具脚本，例如图像分类辅助、模型加密、ncnn混合量化。
- `python/file_handing/`
  零散脚本，例如cv中文读取、文件操作。


### `cpp/`

C++ 通用模块与基础算法，主要包含：

- `cpp/yolo/`
  ONNX / NCNN / 全志NPU / RKNN 的推理封装与接口文件
- `cpp/utils/`
  零散可复用图像处理辅助函数

`projects/` 可复用的套件。

### `projects/`

按具体业务问题归档的专项工程，通常是“入口脚本 + 算法实现 + 配置 / 模型”的组合。和上面的 `python/hdy/parse_file/` 相比，更偏向具体任务验证，技术路线也更垂直。

#### `projects/701/`

面向声呐的目标靠近告警检测工程，入口为 `projects/701/main.py`：

- 输入形式：
  以带角度首行的 CSV 作为输入，首行是 beam 对应角度，后续行为声呐灰度矩阵。
- 预处理方式：
  `data_preprocess.py` 中的 `SonarToImage.preprocess()` 会先按量程截取近距离区域，再通过 `MCR` 增强暗目标，对图像做灰度增强后转为三通道，供 ONNX 模型推理。
- 检测模型：
  `main_algorithm.py` 中通过 `YOLOV8ONNX` 加载模型，配置来自 `config.yml`，说明这一工程已经把“模型参数”和“业务告警参数”拆开管理。
- 跟踪方式：
  不是直接逐帧输出，而是先做简单多目标跟踪。每个目标保存历史轨迹，通过前两帧位移做线性预测，再用 IoU 将新检测结果与历史目标关联。
- 业务逻辑：
  目标不是“检测到”就报警，而是结合 y 方向真实距离变化判断是否持续靠近；同时设置起始距离、最小轨迹长度、偏差比例、最小移动距离等规则，减少误报。
- 结果重构：
  输出结果从图像坐标重新映射回“距离 + 角度 + 置信度”。

“检测模型 + 轻量跟踪 + 业务告警规则”的组合工程。

#### `projects/fish_counter/`

鱼类计数原型，重点不是单帧检测，而是连续帧中的“进鱼 / 出鱼”判断：

- 输入形式：
  同样以 CSV 声呐数据为基础输入，首行角度、后续为强度矩阵。
- 成像方式：
  `data_preprocess.py` 把极坐标声呐数据映射为高分辨率扇形图像，并执行归一化、gamma 增强、CLAHE 和伪彩处理，使目标形态更适合 OBB 检测。
- 检测模型：
  `YoloObbOnnx.py` 使用 `onnxruntime` 加载旋转框模型，输出 `(cx, cy, w, h, angle)` 形式的 OBB，后处理包含旋转框 IoU 与旋转 NMS。
- 跟踪方式：
  `main.py` 中自定义了 `TrackObject` 与 `RadarFishOBBCounter`，通过旋转框 IoU 做帧间匹配，并在短时丢检时用历史轨迹外推位置，实现延迟销毁和轨迹连续性维护。
- 方向判定：
  不是只看一帧中心点，而是结合最近若干帧轨迹的整体位移、角度约束、平均体长阈值，以及是否跨越中心线来判断“右进 / 左出”。
- 输出形式：
  既可以输出每帧的实时计数，也可以生成带轨迹、ID、方向标签和置信度的可视化图像，便于人工复核。

“扇形成像 + OBB 检测 + 多目标跟踪 + 方向计数”。

#### `projects/remove_bright_line/`

针对声呐图像亮线伪影的专项修复工程，同时保留了 Python 和 C++ 两版实现：

- 输入形式：
  读取 CSV 中的二维浮点矩阵作为单通道图像。
- 亮线检测思路：
  核心算法先对图像做左右平移差分，构造残差图；再用中位数和 MAD 估计阈值，把异常高亮的细长结构提取出来。
- 形态学与连通域：
  通过腐蚀去掉散点噪声，再做连通域分析，筛选“宽度较窄但高度足够”的候选亮线区域，同时定位主亮线所在的列范围。
- 掩码修整：
  不仅保留主连通域，还会把同一列范围内、与主亮线相关的其他碎片并入，减少断裂。
- 修复方式：
  `enhancedFlipFillWithLocalSmooth()` 会从亮线两侧截取区域，做镜像翻转与线性权重融合，生成待填充像素；最后再做轻量平滑，减弱修复边界。

“异常亮细线检测 + 局部镜像填补”。

#### `projects/remove_ghost/`

基于目标检测的重影修复工程，“检测驱动修复”：

- 模型依赖：
  `GhostRemover.cpp` 中会先初始化 NCNN / YOLO 检测器，并做一次 warmup，减少首次推理抖动。
- 数据流设计：
  主修复流程始终在 float32 图像上进行，只有检测阶段临时转成 8-bit / 3 通道输入给模型，既满足推理要求，又不损失修复时的数值精度。
- 检测目标：
  当前逻辑主要围绕类别 `cy` 展开，先检测可能需要修复的目标框，再将检测框裁剪到图像边界内。
- 跨帧稳定性：
  工程并不完全相信单帧检测结果，而是维护 `cy` 目标轨迹。对检测框做贪心匹配、速度估计和短时预测，只有累计出现次数达到阈值后才真正参与修复，降低偶发误检带来的错误填补。
- 修复区域构造：
  对每个目标框会区分主目标框和更大的待修补框：前者表示应尽量保留的主体，后者表示整体需要镜像填补的区域。
- 修复方式：
  最终通过 `mirrorFill()` 在局部区域执行镜像补偿，达到压制重影、保留主体的效果。

“目标检测筛选候选区域 + 轨迹稳定化 + 局部镜像修补”。

#### `projects/denoise/`

时序建模的降噪原型：

- 输入形式：
  从多帧 CSV 中读取单通道浮点图像，并按顺序处理。
- 配准方式：
  `sonar_remover.cpp` 中通过 `phaseCorrelate` 估计相邻帧平移量，再用 `warpAffine` 把前后帧对齐到当前帧坐标系。
- 时序参考图：
  对齐后构造三帧时序中值 `temporal_median3`，作为更稳健的背景参考图，用来压制随机噪声和瞬时干扰。
- 差分检测：
  当前帧与时序参考图做差，再做阈值化和形态学处理，得到高亮候选区域。
- 目标保留机制：
  为避免把真实鱼目标一并抹掉，工程还会检查前两帧在相同区域内是否存在对应响应，只对时序上不一致、更像干扰的部分做抑制。
- 输出策略：
  结果图仍保留浮点精度，只有在可视化或存盘时才转成 8-bit 伪彩图。

“相邻帧配准 + 时序中值参考 + 差分抑噪 + 目标保留”。

`projects/` 下的工程可以大致分成两类：

- 检测与行为分析类：`701`、`fish_counter`
- 图像修复与质量增强类：`remove_bright_line`、`remove_ghost`、`denoise`

前者强调模型推理、跟踪和业务规则，后者强调图像处理、局部修复和时序稳定性。
